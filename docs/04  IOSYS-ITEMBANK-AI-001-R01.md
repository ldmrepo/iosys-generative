## 1. 주요 변경 사항

**Qwen3-VL-Embedding** (2026년 1월 출시)이 기존 기술 스택 선정에 중대한 영향을 미칩니다.

### 1.1 Qwen3-VL-Embedding 핵심 특징

| 항목 | 내용 |
|------|------|
| **멀티모달 통합** | 텍스트, 이미지, 스크린샷, 비디오를 단일 임베딩 공간에 매핑 |
| **모델 크기** | 2B / 8B 버전 제공 |
| **최대 토큰** | 32K 토큰 |
| **다국어 지원** | 30개 이상 언어 (한국어 포함) |
| **벤치마크** | MMEB-V2 77.8점 (2025년 1월 기준 1위) |
| **라이선스** | Apache 2.0 (오픈소스) |

### 1.2 기존 접근법 vs Qwen3-VL

| 구분 | 기존 접근법 | Qwen3-VL 접근법 |
|------|-------------|-----------------|
| 텍스트 임베딩 | KURE-v1 / BGE-m3-ko | Qwen3-VL-Embedding |
| 이미지 임베딩 | SigLIP / CLIP | Qwen3-VL-Embedding |
| 임베딩 공간 | 텍스트/이미지 별도 → 후처리 결합 | **단일 통합 공간** |
| 검색 파이프라인 | 텍스트 검색 + 이미지 검색 분리 | **통합 멀티모달 검색** |

## 2. 수정된 기술 스택 권장안

### 2.1 임베딩 모델

| 컴포넌트 | 1차 권장 | 대안 |
|----------|----------|------|
| **멀티모달 임베딩** | Qwen3-VL-Embedding-2B | Qwen3-VL-Embedding-8B (고성능) |
| **텍스트 전용 (fallback)** | Qwen3-Embedding | KURE-v1 |
| **Reranker** | Qwen3-VL-Reranker | - |

### 2.2 Qwen3-VL 선정 근거

1. **단일 모델로 텍스트+이미지 처리** → 파이프라인 단순화
2. **통합 임베딩 공간** → 멀티모달 검색 품질 향상
3. **한국어 포함 다국어 지원** → 별도 한국어 모델 불필요
4. **Reranker 연계** → 검색 정확도 2단계 향상
5. **32K 토큰** → 긴 지문 처리 가능

### 2.3 검색 파이프라인 변경

**기존 설계**:
```
문항 → 텍스트 임베딩 (KURE) ──┐
                              ├──▶ 벡터 결합 → 검색
이미지 → 이미지 임베딩 (SigLIP) ─┘
```

**수정된 설계**:
```
문항 (텍스트+이미지) → Qwen3-VL-Embedding → 단일 벡터 → 검색
                                                         │
                                                         ▼
                                              Qwen3-VL-Reranker → 정밀 순위
```

## 5. 구현 예시

```python
from qwen3_vl_embedding import Qwen3VLEmbedder

# 모델 초기화
embedder = Qwen3VLEmbedder(
    model_name_or_path="Qwen/Qwen3-VL-Embedding-2B",
    max_length=8192
)

# 문항 임베딩 (텍스트 + 이미지 통합)
embedding = embedder.encode(
    text="삼각비를 이용하여 AB의 길이를 구하시오.",
    image="path/to/geometry_image.png"
)

# 검색
results = vector_db.search(embedding, top_k=10)

# Reranker로 정밀 순위 조정
reranked = reranker.rerank(query, results)
```
